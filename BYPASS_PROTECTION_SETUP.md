# Настройка Protection Bypass для обхода Vercel Authentication

## ✅ Решение найдено!

В разделе **Deployment Protection** есть опция **"Protection Bypass for Automation"** - это именно то, что нужно!

## Шаг 1: Добавьте секрет в Vercel

1. В разделе **Deployment Protection** найдите **"Protection Bypass for Automation"**
2. Нажмите **"+ Add Secret"**
3. Введите секрет (например, сгенерируйте: `openssl rand -hex 32`)
4. Сохраните

**Важно:** Этот секрет будет доступен как переменная окружения `VERCEL_PROTECTION_BYPASS` в деплоях.

## Шаг 2: Используйте секрет в админке

Секрет можно передать двумя способами:

### Вариант 1: Через query параметр (проще)

В URL добавьте параметр:
```
?x-vercel-protection-bypass=ВАШ_СЕКРЕТ
```

### Вариант 2: Через заголовок (безопаснее)

Но для iframe это сложнее, поэтому используем вариант 1.

## Шаг 3: Обновите админку

Админка автоматически добавит этот параметр в URL iframe.

## После настройки

1. Сайт будет доступен с параметром `x-vercel-protection-bypass`
2. Vercel Authentication будет обойден
3. Токен `admin_token` все равно будет проверяться для дополнительной защиты
